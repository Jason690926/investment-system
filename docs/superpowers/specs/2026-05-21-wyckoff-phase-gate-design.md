# 威科夫派發判定門檻 — 設計文件

> 日期：2026-05-21｜方案：B（程式算結構事實 + prompt 雙閘）

## 一、問題陳述

`ai_analyzer_v2.py`（目前在用的 v2）的威科夫月K相位**完全由 AI 自由判斷**，沒有任何量化錨點（舊的 `wyckoff.py` v2 未使用）。結果是系統性偏空：

- 5/20 報表 15 支判 10 支放空，5/21 隔日 14 支上漲。
- 多月大漲股（臻鼎 +200%、合晶五連陽、瑞軒、南亞科）只因「最近一個月收陰 + 量縮」就被打成「派發 / 假突破 / 頂部結構」。
- 使用者校準基準：**多月大漲後單月收陰且量縮，相位仍是「上漲」，方向 long**（屬上漲中的回測，非派發）。

## 二、設計目標

修掉「false 派發」——在 prompt 加入一道**客觀結構閘**，當月線結構未轉弱時，硬禁止 AI 標派發 / 再派發 / 下跌。

### 非目標

- 不改本間宗久（K線確認層）、李佛摩（時機層）邏輯。
- 不改 P&F 目標、量能門檻計算。
- 不重寫或啟用舊 `wyckoff.py`（v1 模組保留不動）。
- 不阻擋多方相位（積累 / 再積累 / 上漲）——本案只收緊空方誤判，不對稱。

## 三、【月線結構客觀事實】欄位定義

新增 helper `compute_monthly_structure(monthly_bars, weekly_bars, price, ma60)`，從既有 `enriched_data` 算出，回傳 dict。月K 12 根、週K 26 根皆已存在於 `enriched_data`。

**「已收盤」定義**：每組 bars 的最後一根視為進行中（current month/week），結構判定一律用其前 N 根已收盤 bar，即 `bars[-4:-1]`。

| 欄位 | 算法 | 值域 |
|------|------|------|
| `monthly_structure` | 最近 **3 根已收盤**月K的高點、低點序列 | 升 / 跌 / 轉折 / 橫 |
| `consecutive_bear_months` | 從最新已收盤月K往回數，連續 close<open 的月數 | 0、1、2… |
| `drawdown_from_peak` | (近12月最高收盤 − 現價) ÷ 最高收盤 × 100 | %（可為負，代表現價創高）|
| `price_vs_ma60` | 現價 vs 日線 MA60 | 在上 / 在下 / 未知 |
| `structure_flag` | 綜合上列，見下方判定 | 結構未轉弱 / 結構轉折中 / 結構已轉弱 |
| `weekly_momentum` | 最近 **3 根已收盤**週K結構（唯讀，不參與閘門）| 升 / 跌 / 橫 |
| `weekly_hold_support` | 最近 3 根週K收盤群聚且墊高（唯讀）| 是 / 否 |

### `monthly_structure` 判定（3 根，舊→新 = b1,b2,b3）

- **升** = 高點 h3≥h2≥h1 **且** 低點 l3≥l2≥l1
- **跌** = 高點 h3≤h2≤h1 **且** 低點 l3≤l2≤l1
- **轉折** = 高點與低點只有一邊單調（一邊墊高、一邊下降）
- **橫** = 其餘（高低點皆非單調）

### `structure_flag` 判定（依序，先符合先回傳）

```
結構已轉弱  ← price_vs_ma60=在下  或  monthly_structure=跌  或  consecutive_bear_months≥2
結構未轉弱  ← price_vs_ma60=在上  且  monthly_structure∈{升,橫}  且  consecutive_bear_months≤1
結構轉折中  ← 其餘（主要為 monthly_structure=轉折 且現價在 MA60 上）
```

### `weekly_hold_support` 判定（唯讀，供李佛摩時機層用）

- 最近 3 根已收盤週K收盤離散度 (max−min)/mean < 3%
- **且** 最新一根週K收盤 ≥ 前兩根週K收盤
- 兩者皆成立 → 是

> 說明：刻意採「收盤」基準而非「最低點」。臻鼎週K收盤 401/401/408.5（離散 1.9%、墊高）= 守穩=是；盤中低點 372 的下影線交給 AI 量價層判讀（收回支撐屬偏多）。

## 四、Prompt 雙閘規則

### 結構閘（硬護欄，注入 `static_block` 的「結構方向判定」段）

| `structure_flag` | 允許的威科夫相位 | 派發 / 再派發 / 下跌 |
|------------------|----------------|----|
| `結構未轉弱` | 上漲 / 再積累 / 不明 | **🚫 硬禁止** |
| `結構轉折中` | 不明 / 再積累 / 派發 | ⚠️ 可標，須附量價證據 |
| `結構已轉弱` | 派發 / 下跌 / 再派發 / 不明 | ✅ 允許（仍須量價證據）|

prompt 明文：「**若【月線結構客觀事實】的結構旗標為『結構未轉弱』，禁止標派發 / 再派發 / 下跌**，威科夫相位只能在『上漲 / 再積累 / 不明』之中選。結構旗標由程式計算，禁止 AI 推翻。」

### 量價證據閘（語意判斷，留給 AI）

即使結構閘允許派發，AI 必須在「一、威科夫骨幹分析」明確列出至少一項**具體派發訊號**，不得僅憑「單月收陰 + 量縮」：

- 高位量增不漲（放量卻收長黑 / 長上影）
- 高位放量衝頂後量價背離
- 跌破前波明顯低點且伴隨放量

**新增正向型態（平衡偏空慣性）**：

- 價格連續測試同一支撐（`weekly_hold_support=是`）+ 回測時量縮 + 反彈時量增 → 屬**吸籌 / 再積累的次級測試（SOT）**，偏多，禁標派發。
- 同型態但回測時放量 → 支撐遭消耗，偏空成立。

## 六、測試（pytest，`tests/`）

- `monthly_structure` 四種值各一案例（升 / 跌 / 轉折 / 橫）。
- 臻鼎型 fixture（月K 升、陰線 1、現價在 MA60 上）→ `結構未轉弱`。
- 晶心科型 fixture（月K 跌、現價跌破 MA60）→ `結構已轉弱`。
- 連 2 月陰線 → `結構已轉弱`（即使現價在 MA60 上）。
- 進行中月份正確排除（`bars[-4:-1]`）。
- `weekly_hold_support`：收盤 401/401/408.5 → 是；收盤遞減 → 否。
- 注入文字含「結構旗標」「禁止標派發」字樣。

## 七、驗收標準

1. pytest 全綠（既有 158 + 新增）。
2. 燒 ~$0.6 重跑一鍵分析後：
   - 臻鼎、合晶、瑞軒、南亞科（月K結構=升、現價在 MA60 上）**不再出現「派發 / 再派發」字樣**，相位落在上漲 / 再積累。
   - 晶心科、創惟（真實下跌結構）**仍可標空方相位**——驗證閘門不對稱、未誤殺真派發。
3. 報表中凡 `結構未轉弱` 的股，威科夫相位不得為派發 / 再派發 / 下跌。
